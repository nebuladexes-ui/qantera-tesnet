# Qantera Documentation Style

## Voice

Use a technical, direct, crypto-native voice:

- precise rather than promotional;
- confident without absolute security claims;
- action-oriented for tutorials;
- explicit about testnet status and limitations.

## Preferred terminology

Use:

- quantum-resistant;
- post-quantum cryptography;
- crypto-agile;
- EVM-compatible execution;
- validator consensus;
- transaction finality;
- on-chain;
- smart contract;
- JSON-RPC;
- testnet token.

Avoid:

- quantum-proof;
- impossible to hack;
- military-grade;
- guaranteed security;
- risk-free.

## Page pattern

Each tutorial should include:

1. Goal
2. Prerequisites
3. Steps
4. Expected result
5. Troubleshooting
6. Next steps
