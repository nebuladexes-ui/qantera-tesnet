# Monitoring

Validators must be monitored continuously to detect missed duties and infrastructure problems.

### Important metrics

Validator operators should monitor:

* validator status;
* node synchronization;
* latest block;
* consensus participation;
* proposed blocks;
* missed duties;
* peer count;
* CPU usage;
* memory usage;
* disk usage;
* network connectivity;
* validator software version.

### Recommended alerts

Create alerts for:

* validator offline;
* node not synchronized;
* block height stopped;
* repeated missed duties;
* low peer count;
* disk almost full;
* high memory usage;
* validator-key errors;
* outdated software version.

### Operational checks

Operators should regularly confirm:

1. The node is running.
2. The node is synchronized.
3. The validator is active.
4. The server has sufficient disk space.
5. Monitoring alerts are working.
6. The latest required software version is installed.
