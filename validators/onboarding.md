# Onboarding

Public validator onboarding has not yet started.

The official onboarding process will be published before validators are allowed to join.

### Expected onboarding flow

The process may include:

1. Review validator requirements.
2. Prepare the required server.
3. Install the official Qantera node client.
4. Synchronize the node.
5. Generate validator keys.
6. Register or apply as a validator.
7. Wait for activation.
8. Confirm validator status.
9. Configure monitoring and alerts.

### Information that will be published

The final onboarding guide will provide:

* official software links;
* installation commands;
* validator registration process;
* required contract addresses;
* key-generation commands;
* activation rules;
* validator status tools;
* exit procedures.

### Safety warning

Qantera validator onboarding will never require operators to:

* share a seed phrase;
* share a private key;
* send funds to a personal wallet;
* install software from an unofficial source;
* provide remote server access to an unknown person.
