# Qantera GitBook Content Map

The information architecture follows a practical developer journey:

1. Understand Qantera Public Testnet
2. Add the network to an EVM wallet
3. Verify chain ID and RPC access
4. Claim QTER after the faucet opens
5. Send transactions
6. Build and deploy Solidity contracts
7. Use JSON-RPC and Web3 SDKs
8. Prepare node and validator infrastructure
9. Understand the post-quantum security model
10. Join contributor programs
11. Troubleshoot and follow releases

The documentation deliberately:

- keeps standard Ethereum RPC method names such as `eth_*`;
- documents Qantera as an EVM-compatible Layer 1 public testnet;
- does not copy proprietary virtual machines, bridges, or cross-signing systems from reference projects;
- adds Qantera-specific Social Tasks, Yap Tasks, Genesis NFT, and leaderboard sections;
- separates confirmed public configuration from unpublished protocol details;
- treats post-quantum security as an end-to-end protocol specification, not only an algorithm name.
