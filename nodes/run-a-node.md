# Run a node

Public node software is not yet available.

This page will be updated when Qantera releases the official node client.

### Information that will be provided

The public node guide will include:

* supported operating systems;
* minimum hardware requirements;
* recommended hardware;
* installation files;
* Docker image;
* genesis configuration;
* bootnodes;
* required ports;
* startup commands;
* synchronization options;
* snapshot information;
* upgrade procedures.

### Before running a node

Future node operators should prepare:

* a dedicated server;
* a stable internet connection;
* sufficient storage;
* system monitoring;
* secure firewall rules;
* regular backups.

### Security reminders

When node software becomes available:

* download it only from an official Qantera source;
* verify release checksums;
* do not expose administrative APIs publicly;
* restrict RPC access when running a private node;
* keep the operating system updated;
* never share node or validator keys.

> Official installation commands will be published after the Qantera node client is released.
