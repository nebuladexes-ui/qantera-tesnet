# Upgrades

Node software must be kept up to date to remain compatible with the network.

### Before upgrading

Node operators should:

1. Read the official release notes.
2. Confirm the required upgrade version.
3. Verify the release checksum.
4. Back up configuration files.
5. Check available disk space.
6. Prepare a rollback plan.
7. Test the upgrade on a non-critical node when possible.

### During the upgrade

1. Stop the node safely.
2. Install the official release.
3. Keep the existing blockchain data unless instructed otherwise.
4. Restart the node.
5. Confirm synchronization.
6. Check RPC and peer connectivity.

### After upgrading

Verify:

* node version;
* latest block;
* synchronization status;
* peer count;
* RPC availability;
* system resource usage.

### Important

Only follow upgrade instructions published through official Qantera channels.

The release-notes page and node-upgrade commands will be added when the public node client becomes available.
