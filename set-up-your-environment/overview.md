# Overview

Follow this section to connect your wallet to Qantera Public Testnet and prepare for your first transaction.

### What you need

You need:

* an EVM-compatible wallet;
* a separate account for testnet activities;
* the official Qantera network information;
* QTER test tokens for gas.

Supported wallet options include:

* [MetaMask](https://metamask.io/)
* [OKX Wallet](https://web3.okx.com/)
* [Rabby Wallet](https://rabby.io/)
* other wallets that support custom EVM networks.

> Use a separate wallet for testnet activities. Do not use an account that holds valuable mainnet assets.

### Qantera network information

For the latest RPC endpoints, chain ID, native token, and explorer information, see [**Network Details**.](../introducing/network-details.md)

### Setup process

1. Install an EVM-compatible wallet.
2. Create or select a testnet account.
3. Add Qantera Public Testnet.
4. Confirm that the chain ID is `974621`.
5. Request QTER when the faucet becomes available.
6. Send a small test transaction.
7. Check the transaction on the [Qantera Explorer](https://explorer.qantera.network/).

### Choose your wallet

Continue with:

* **Add Qantera to MetaMask**
* **Add Qantera to OKX Wallet**
* **Rabby and Other EVM Wallets**

### Safety reminders

* Never share your seed phrase or private key.
* Verify that the RPC domain ends with `qantera.network`.
* Confirm chain ID `974621` before signing.
* Never pay anyone for QTER test tokens.
* Verify transactions through the [Qantera Explorer](https://explorer.qantera.network/).
