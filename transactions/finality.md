# Finality

Finality describes when a transaction is considered permanently accepted by the network.

### Inclusion and finality

These terms are different:

#### Block inclusion

The transaction has been added to a block.

#### Finality

The network has reached the point where the block is not expected to be reversed.

### What users should check

After sending a transaction:

1. Open the [Qantera Explorer](https://explorer.qantera.network/).
2. Search for the transaction hash.
3. Confirm the transaction status.
4. Confirm the block number.
5. Wait for the required confirmations before treating the action as complete.

### Current status

The official Qantera consensus and finality model has not yet been published.

Until those details are available, applications should avoid displaying a fixed finality time.

The documentation will be updated when the following information is confirmed:

* consensus mechanism;
* expected block time;
* expected finality time;
* recommended confirmation count.
