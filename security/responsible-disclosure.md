# Responsible disclosure

Security researchers are encouraged to report potential vulnerabilities responsibly.

### What to include

A useful report should include:

* affected component;
* clear description;
* steps to reproduce;
* expected behavior;
* actual behavior;
* possible impact;
* proof of concept, when safe;
* suggested fix, when available.

### Please avoid

Do not:

* access user data;
* steal or move assets;
* disrupt network availability;
* publicly disclose an unpatched issue;
* use social engineering;
* test against systems outside the approved scope.

### Reporting channel

The official security contact and disclosure policy will be published before the public bug-bounty program begins.

Until then, do not send sensitive vulnerability details through public social channels.

This page will be updated with:

* security email;
* bug-bounty scope;
* response timeline;
* safe-harbor rules;
* reward conditions.
